# ensure-template-configs.py

## Overview

`ensure-template-configs.py` is a Python replacement for the original [`templates/config.sh`](archive/config.sh) script. It converts JSON-based Packer configuration files into HCL2 format while maintaining full compatibility with the legacy behavior.

This script enables modern Packer HCL2 usage while keeping configuration data in clean, portable JSON format for easier consumption by CI/CD pipelines, Jenkins, etc.

## Background & Motivation

Prior to 2022, all Packer templates used pure JSON. This approach had several limitations:
- No dynamic value injection (`templatefile()` support)
- Sensitive data (passwords, SSH keys) exposed in source control
- Poor support for complex data structures (lists, maps)

The solution was a hybrid approach:
- Maintain configuration in **JSON** (for tooling compatibility)
- Convert to **HCL2** at build time using dynamic expressions (`%%var.*`, `%%local.*`)
- Use `packer hcl2_upgrade` + post-processing to produce final `.pkr.hcl` and `.pkrvars.hcl` files

## Features

- **Incremental conversion** using a single `.conversion-state.json` cache (only processes changed files)
- **Exact parity** with original `config.sh` behavior (including `%%` pattern handling)
- Proper handling of `vars`, `vardef`, and `build` file types
- Symbolic linking of common HCL files into distribution directories
- Flexible logging (`-L` / `--log-level`)
- Support for forcing full rebuilds and optional backups
- Clean separation between distribution-level and version-level files

## Usage

```bash
# Normal run (INFO level)
python3 ensure-template-configs.py

# Debug mode
python3 ensure-template-configs.py -L DEBUG

# Force full conversion of all files
python3 ensure-template-configs.py --force

# Specific distributions only
python3 ensure-template-configs.py Ubuntu,24.04 Ubuntu,26.04

# Enable backups
python3 ensure-template-configs.py --backup
```

Then perform packer validation on platform configurations (same validation performed in github actions):
```shell
$ cd templates
$ packer validate -only vsphere-iso.* \
  -var-file=env-vars.PROD.json.pkrvars.hcl \
  -var-file=Ubuntu/distribution-vars.json.pkrvars.hcl \
  -var-file=Ubuntu/26.04/server/template.json.pkrvars.hcl \
  -var-file=Ubuntu/26.04/server/box_info.small.json.pkrvars.hcl \
  -var vm_template_build_name=vm-template-ubuntu26.04-small-prod-00006 \
  -var vm_template_build_type=small \
  -var vm_template_name=vm-template-ubuntu26.04-small-prod \
  -var vm_build_env=PROD \
  -var iso_dir=Ubuntu/26.04 \
  -var iso_file=ubuntu-26.04-live-server-amd64.iso \
  Ubuntu/

$ packer validate -only vsphere-iso.Windows/server \
  -var-file=env-vars.PROD.json.pkrvars.hcl \
  -var-file=Windows/server/distribution-vars.json.pkrvars.hcl \
  -var-file=Windows/server/2016/template.json.pkrvars.hcl \
  -var-file=Windows/server/2016/box_info.standard.json.pkrvars.hcl \
  -var vm_template_build_name=$TEMPLATE_BUILD_ID \
  -var vm_template_build_type=standard \
  -var vm_template_name=$TEMPLATE_BUILD_ID \
  -var vm_build_env=PROD \
  Windows/server/
```

### Options

| Option            | Description                          |
|-------------------|--------------------------------------|
| `-L, --log-level` | "Set log level (DEBUG, INFO, etc.)"  |
| `--clear-cache`   | .Clear conversion state cache        |
| `--force`         | Force conversion of all files        |
| `--backup`        | Create .bak files before overwriting |


## File Handling Rules

- Distribution level: distribution-vars.json, build-config.json
- Version level: box_info.*.json, template*.json
- Common files (common-vars.json, etc.) are handled specially
- Excludes: archive/, save/, backup/, etc.

## Post-Processing Steps (matching original shell script)

1. Run `packer hcl2_upgrade` for build files
2. `$$` → `$`
3. `\"` → `"`
4. `%%...` → `...` (critical for dynamic expressions)
5. autogenerated_123 → platform name (e.g. Ubuntu)

This script maintains full backward compatibility while being significantly more maintainable and efficient.
